# 📖 小D (Creative Director) 安裝與使用說明手冊

本手冊引導使用者如何透過 GitHub 安裝小D 技能，並詳細說明 AI 運作步驟與「人機協作（Human-in-the-loop）」的互動閘門。

---

## 🛠️ 第一部分：透過 GitHub 安裝與開始使用

### 1. 複製技能倉庫 (Clone Repository)
請在您的專案根目錄下，切換至 AI 助理的技能目錄，並複製本倉庫：

```bash
cd [您的專案根目錄]/.agents/skills/
git clone https://github.com/uc94jooo/creative-director.git
```

### 2. ⚠️ 系統載入路徑重要提醒 (Folder Structure Warning)
本倉庫在 GitHub 上安排的結構中，主程式碼收納在子資料夾 `creative-director/` 內。其物理目錄結構如下：
```
.agents/skills/creative-director/
├── creative-director/          <── 主程式碼資料夾
│   ├── SKILL.md
│   ├── references/
│   └── scripts/
└── johs-notes/                 <── 思考與說明文件
```
*   **載入警告**：如果您的 AI 代理人系統（如 Claude Code、Cursor 等）預設只會掃描一層目錄，請確保您的設定將小D 載入路徑指向內層的 `.agents/skills/creative-director/creative-director/`，或者直接將內層 `creative-director/` 下的所有檔案搬移至倉庫根目錄下，否則系統可能會因為找不到 `SKILL.md` 而無法啟動技能。

### 3. 環境準備與初始化
在您的專案根目錄下，建立一個名為 `專案資料區/` 的資料夾。
*   （若忘記建立，小D 在執行 Phase 0 時偵測不到，會自動為您在根目錄建立空的 `專案資料區/` 檔案夾並暫停 Turn 等待您放入檔案）。

### 4. 啟動對話 (Triggering the Skill)
在與 AI 的對話中，只要輸入以下任一口語指令，即可自動觸發 Phase Router 進入小D 的工作流：
*   *「小D看案子」*
*   *「幫我開新專案」*
*   *「小D，有新 brief」*

## 🤝 第二部分：AI 工作步驟與人機協作互動

小D 的核心邏輯是以**「資料為引導的雙向協作」**，在策略對齊階段（Phase 0-2）共有 **10 個步驟骨架**，其中包含 **2 個關鍵的人工參與閘門**：

### 1. 策略段（Phase 0 - 2）的 10 步驟骨架

```
[Phase 0]
  步驟 1：掃描專案資料區 (防呆阻斷 ── 人工選擇專案子資料夾)
  步驟 2：建立隱藏軌 .cd-notes/ (靜默執行)
  步驟 3：列出專案檔案 (防呆阻斷 ── 人工選擇要讀取的檔案)
  步驟 4：自動標準化建檔整合 MD (背景靜默執行)
[Phase 1]
  步驟 5：前 6 點限制條件萃取 (自動執行)
  步驟 6：資訊充足度診斷 (🚨 人工參與閘門一：補料防呆與能力校準)
[Phase 2]
  步驟 7：大腦隱性策略推演 (品牌原型/角色鑑定/空雨傘 ── 背景執行)
  步驟 8：對齊表自我診斷 (4避坑防線檢查 ── 自動執行)
  步驟 9：輸出 13 點策略對齊整合表 (自動執行)
  步驟 10：Checkpoint 1 策略確認 (🚧 人工參與閘門二：戰略對齊與深挖)
```

---

### 2. 核心人工參與閘門詳解

#### 🚨 閘門一：Phase 1 補料診斷與能力校準 (非強制，視資料完整度觸發)
當小D 掃描完您選取的 Brief 檔案後，如果發現關鍵資訊（受眾、競品、預算等）空泛或缺少，會在此處**強制暫停對話**，拋出 3-5 個具策略針對性的問題。

這是小D 判定您「能力程度」並進行**資料動態萃取**的黃金時機：
*   **狀況 A：您盡力且完整地回答了問題 (Rich Input)**
    *   *AI 行為*：小D 判定資料充足且協作水準高，會進入**「預設模式」**，在 Phase 2 直接為您產出銳利硬核的策略對齊表。
*   **狀況 B：您回答簡短或資料極度模糊 (Shallow Input)**
    *   *AI 行為*：小D 會啟動**「問答框架引導 (Guided Extraction)」**，暫緩進入 Phase 2。小D 會調用微型工具（如 **JTBD 任務雇用論**或**競品體感缺陷對標法**）以二選一、情境題的方式，像教練一樣輔導您把更深層的背景細節擠出來，直到補齊高質量資料為止。

#### 🚧 閘門二：Phase 2 Checkpoint 1 策略對齊表確認 (強制觸發)
在小D 生成 13 點「策略對齊整合表」後，會**立刻終止 Turn 並煞車暫停**，等待您的授權與回饋：
*   **路徑 A：確認通過 (Proceed)**
    *   *人工動作*：輸入「確認」或「Proceed」。
    *   *AI 行為*：小D 將此版本存檔為實體檔案，並自動無縫推進至 Phase 3 創意發想。
*   **路徑 B：策略辯論與深挖 (Correction Loop)**
    *   *人工動作*：對受眾、痛點、單一主張（SMP）或風格提出質疑或追問。
    *   *AI 行為*：小D 會在此時默默校準您的協作人格（如老手、執行者、探索者、忙碌決策者），並**強制啟用「三階深度線頭系統 (Breadcrumb Bank)」**，從題庫中選擇一階（邏輯挑戰）、二階（創意轉譯）或三階（未來預判）的合法問題引導討論，直到對齊表修改至您滿意並給出 Proceed 訊號為止。

## 💾 第三部分：隱藏軌與存檔規範（背景機制）

在人機互動過程中，小D 為了確保跨 Session 與跨輪次的記憶一致，會自動在背景寫入兩種隱藏檔（存於 `.cd-notes/` 隱藏資料夾下，不主動對您提及）：

1.  **`insight-pass-0N.md` (表的變化軌)**：
    *   每次對齊表內容有任何新增、修正、推翻時「即時寫入」，記錄大腦框架（原型、空雨傘）的推演軌跡。
2.  **`dialogue-log-0N.md` (全互動史的週期帳)**：
    *   每次 Checkpoint 通過或評估結束時「一次性寫入」，記錄 AI 詢問、USER 回答、AI 思考、USER 反饋的對話摘要，並沉澱下階段的判斷標準。
