# 📖 小D (Creative Director) 安裝與使用說明手冊

這是一份給行銷企劃、創意人員（非工程背景）使用的簡單安裝與運作流程手冊。

---

## 🛠️ 第一部分：如何開始使用 (安裝)

### 1. 下載小D 檔案
*   請到 GitHub 點選 **Download ZIP** 下載本專案的壓縮檔，並在電腦上解壓縮。
*   解壓縮後，你會得到一個名為 `creative-director` 的資料夾。

### 2. 上傳到你的 AI 軟體
*   開啟你的 AI 軟體（例如 **Claude Projects** 專案區、**Cursor** 或 **Gemini** 等）。
*   在專案的「知識庫 (Add Content)」或「技能/指令設定」中，直接把 `creative-director` 資料夾裡的檔案（特別是 `SKILL.md`）拖曳上傳進去。
*   *(備註：小D 的主程式碼都在 `creative-director/` 子目錄內，請確保 AI 有讀取到裡面的 `SKILL.md`。)*

### 3. 開口叫醒小D
*   在專案中開啟一個新的對話視窗。
*   直接輸入：**「小D看案子」** 或是 **「幫我開新專案」**，小D 就會醒來為你服務！

---

## 🤝 第二部分：專案運作流程說明

小D 在處理新案子時（從準備資料到策略拍板），會依序執行以下步驟：

### 1. 開始前：自動檢查更新
*   偵測到新版本時會詢問你是否要更新。
*   不更新也完全不影響這次的案子。

### 2. 準備資料
*   **告訴小D 這次案子放在哪個資料夾 (需要你配合)**
    *   在「專案資料區」開一個新資料夾，把 Brief、簡報、逐字稿丟進去。
    *   或是直接在對話裡貼網址與文字。
    *   還沒有資料夾的話，小D 會先自動幫你建一個。
*   **挑選這次要用的檔案 (需要你配合)**
    *   小D 會列出資料夾中的檔案讓你挑選。
    *   可以一次勾選多個檔案，小D 不會自己亂猜。
*   **整理成好讀的規劃筆記**
    *   把零散資料統整成一份文件方便日後查閱。
    *   完成後會告知你檔案位置並自動往下走。

### 3. 釐清方向
*   **初步整理案子的大方向**
    *   釐清要解決的任務、預算與預期成果。
    *   列出時間、通路、品牌地雷等限制。
    *   分析競爭對手在消費者體驗上的具體破綻。
*   **資料不具體時停下來問你 ( need human interaction )**
    *   如果出現「所有人」、「所有品牌」等籠統描述，小D 不會自己腦補。
    *   小D 會提出 3 至 5 個問題，等你回答完才繼續。
*   **判斷這次案子的份量大小**
    *   區分是一則貼文、一檔活動，還是整個品牌重塑。
    *   確保評分創意時抓對規模，資源分配不跑偏。
*   **多檢查一次：這件事值不值得做**
    *   僅在辦活動或開發小工具等「真實體驗」案子觸發。
    *   思考拿掉廣告後是否真能幫到人。
    *   思考無抽獎誘因下是否有人願意分享。
    *   評估是短期噱頭還是可長期的服務。

### 4. 找到洞察
*   **挖掘受眾心理與品牌定位**
    *   找出立體的品牌個性、受眾糾結與未被滿足的期待。
    *   提防將受眾非黑即白二分法。
    *   不把「大家愛看的內容形式」誤認成「買單理由」。
    *   不漏掉中間搖擺的客群、不用CP值等空話帶過。
*   **整合為一張策略對齊表**
    *   包含四大塊：任務限制、品牌連結、受眾痛點、洞察方向。
    *   收攏所有推演成果，方便一眼看懂全貌。
*   **請你確認對齊表 ( need human interaction )**
    *   小D 會停下來等待你確認，不自己衝去想創意。
    *   避免策略理解錯誤導致後續白做工。
*   **支援多輪調整與比對 ( need human interaction )**
    *   你可以一輪輪修改，小D 會標示出新舊版本差異。
    *   整理成完整新版，不用在歷史紀錄中自己拼湊。
*   **定案存檔並暫停**
    *   拍板定案策略表並保存檔案。
    *   是否啟動下一步的創意發想，完全等你指令才開始。
