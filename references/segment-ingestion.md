# Phase 0: INGESTION (原始資料導入與工作區設定)

> 本檔為 Phase 0 專案目錄初始化、檔案導入與背景建檔的完整操作細節。
> 進入 Phase 0 的第一動作＝Read 本檔。
> 結案指標＝完成專案子目錄與 `[YYYY-MM-DD]_[專案名稱]整合規劃.md` 建檔，靜默建立 `.cd-notes/`，並自動無縫推進至 Phase 1。

## 1. 工作區路徑與防呆掃描 (Directory Scan)

本階段任務是自動化專案資料的導入與標準化結構建檔。被觸發時小D 必須立即執行，嚴禁使用籠統話術（如「請給我案子資料」）跳過：

1. **工作區路徑與防呆掃描**：
   - 取得本 `SKILL.md` 檔案的實體路徑，並往上追溯，定位出與 `.agents/`（或 `.claude/`）同層的「專案根目錄」。
   - 💡 **路徑對照範例**：
     - 若 `SKILL.md` 位於：`[專案根目錄]/.agents/skills/creative-director/SKILL.md`
     - 則其專案根目錄為：`[專案根目錄]/`
     - 應掃描的 `專案資料區/` 應與 `.agents/` 同層，即位於：`[專案根目錄]/專案資料區`
   - **替身 (Alias) 與路徑對照處理**：
     - 若工作區下的 `專案資料區` 或 `NOTES` 是 macOS 替身 (Alias) 檔案，小D 必須讀取工作區的 `AGENTS.md`（或 `README.md`）中的「📂 外部路徑對照與替身 (Alias) 規則」取得其指向的實際目標路徑，以該目標路徑作為實體掃描與讀寫的目錄。這樣可避免在技能檔中寫死個人絕對路徑，確保不同使用者的跨平台相容性。
   - ⚠️ **防混淆**：`專案資料區/` 與專案根目錄外層可能存在的 `_Project/`（跨專案共用資料夾，其他工具在用）是完全不同的兩個資料夾，嚴禁混淆或誤植。只認與 `.agents/`（或 `.claude/`）同層的 `專案資料區/`。
   - **狀況 A：不存在實體目錄且無法對照**
     - 立即在專案根目錄下建立空的 `專案資料區` 資料夾。
     - **必須立刻終止 Turn**，輸出以下訊息後暫停，等待使用者放置檔案（請將路徑動態替換為實際定位到的絕對路徑）：
       > 「已為您在工作區建立 [專案資料區](file:///[實際專案根目錄的絕對路徑]/專案資料區)。請將該案相關的 Brief、簡報或檔案放入該目錄中，或直接在對話中貼上資料。」
   - **狀況 B：存在實體目錄（或已成功對照至外部目標路徑）**
     - 讀取並列出該目錄下的所有子資料夾（以通用形式例如 `專案A-2026-03/`、`客戶A-2023/` 列出）。
     - **必須立刻終止 Turn**，列出子資料夾清單，並詢問：
       > 「偵測到以下專案資料夾，請問您要分析哪一個？或是否要建立新案子？（請提供名稱，我會為您建立子目錄）」
       > *（嚴禁代選或自行假設，必須等待使用者明確指示）*

2. **資料來源確認與多選**：
   - 使用者選定子資料夾（如 `專案A-2026-03/`）後，掃描該目錄內所有檔案。
   - 列出所有可用的原始檔案（含 PDF、Markdown、PPT 等）與線上文件網址。
   - **再次終止 Turn**，詢問：
     > 「請問此案要分析以下哪些檔案？（支援多選，例如：1, 3, 5）若有其他線上網址或對話檔案，請一併提供。」
     > *（嚴禁代選，必須等待使用者明確指示）*


## 1.5 影片素材前處理 (Video Pre-processing)

若使用者選定的檔案中含有本機影片檔（`.mp4`、`.mov` 等），**在進入下方「運作機制與背景建檔」之前**，必須先呼叫 `video-transcript` 技能：
- 依 `video-transcript` SKILL.md 的首次使用引導流程執行（說明用途、列時間/風險、取得同意後才跑）。
- 技能執行完畢後，取得的逐字稿文字與關鍵幀畫面讀圖結果，視為該影片檔案的「原始內容」，一併納入下方建檔流程的來源。
- **不得**只用逐字稿當唯一來源就建檔——關鍵幀讀圖若讀到畫面疊字（如成效數據、標題卡），必須寫入「核心變數備忘錄」對應欄位，不可遺漏。
- 純網址（YouTube 等線上影片）不在 `video-transcript` 處理範圍內，維持原本網址讀取流程。

## 2. 運作機制與背景建檔 (Ingestion Background Processing)
當使用者選定檔案後，小D 必須在後台執行以下標準化整合建檔與隱藏軌建立工作，全程不可中斷對話：

1. **標準化 Markdown 整合建檔**：
   - **關聯性與增量整合判斷**：讀取所選文件前，必須判斷其關聯性。若選取了多個檔案（例如：一個既有的專案主規劃 MD 檔與其他新增的 PDF/網址簡報），**嚴禁分開建立多個碎片化的 MD 檔**。
   - **整合規則**：
     - 若該專案資料夾中已存在主規劃 MD 檔，小D 必須將新增的資訊「增量合併」寫入該主 MD 檔中（此時目標寫入路徑必須為經 `AGENTS.md` 解析後對照的實體路徑）。
     - 若無既有 MD 檔，則應將所有選定檔案的資訊整合成「單一標準 Markdown 檔案」輸出。
   - **檔名命名規則**：`[YYYY-MM-DD]_[專案名稱]整合規劃.md`。
   - **檔案內容結構**：必須符合下方模板，完成資訊的結構化與初步切片：

```markdown
---
source01: "[第一個原始檔名或網址]"
source02: "[第二個原始檔名或網址，若有複數檔案依此類推：source03、source04...]"
import_date: [YYYY-MM-DD]
category: "Brief / Background"
tags: [[標籤1], [標籤2]]
---

# [專案名稱/主題]

## 1. 原始內容文字大綱 (Raw Content Outline)
[若有多個來源，請以 ### 部分一：[來源A名稱]、### 部分二：[來源B名稱] 分段紀錄各檔案之核心文字大綱，剔除贅詞與格式代碼。若為單一來源則直接撰寫大綱。]

## 2. 核心變數備忘錄 (Core Variable Notes)
[將所有選定來源的關鍵變數合併收攏至此處統一撰寫：]
- 品牌與品類背景：
- 目標受眾與痛點初步描述：
- 預算與管道限制：
- 主要競爭對手：
```

2. **隱藏軌目錄建立 (`.cd-notes/`)**：
   - 在專案子資料夾的實體指向路徑下建立名為 `.cd-notes/` 的隱藏目錄。
   - 此目錄專門用於靜默儲存小D 的原始推演痕跡（例如 Phase 2 的品牌原型鑑定、角色鑑定、空雨傘原始推演過程）。
   - 全程不向使用者提及此目錄的存在與內容。


## 3. 建檔確認回報與自動推進
1. 建檔完成後，向使用者回報主規劃 MD 檔案的路徑（必須使用 `file:///` 格式的實體解析路徑，但絕對不提 `.cd-notes/`）。
2. 回報後**自動、無縫地進入 Phase 1**。第一動作為載入 `[[references/segment-strategy.md]]`，開始執行需求解讀與限制判定。
