# Phase 0: INGESTION (原始資料導入與建檔)

## 運作機制 (Ingestion Flow)
當使用者請小D看案子時，小D 必須先執行以下互動式建檔流程：

1. **掃描並列出專案資料夾**：
   - 優先掃描根目錄下的 `[專案資料區](專案資料區)` 資料夾，並列出底下所有的子資料夾（如 `Mia C'bon2023`、`Mia C'bon2026`、`測試客戶` 等）。
   - 主動詢問使用者要看哪一個專案資料夾，並等待使用者選擇。

2. **列出檔案並提供多選**：
   - 使用者選定專案資料夾後，列出該資料夾內的所有檔案與線上文件（若有提供網址也在此步驟對齊）。
   - 請使用者選擇要分析哪些檔案（明確告知可以多選），並等待使用者回覆。

3. **標準化 Markdown 整合建檔**：
   - **關聯性與增量整合判斷**：小D 讀取所選文件前，必須判斷其關聯性。若使用者選取了多個檔案（例如：一個既有的專案主規劃 MD 檔與其他新增的 PDF/網址簡報），**嚴禁分開建立多個碎片化的 MD 檔**。
   - **整合規則**：
     - 若資料夾中已存在該案的主規劃 MD 檔，小D 必須將新增的 PDF/文件資訊「增量合併」寫入該主 MD 檔中。
     - 若無既有 MD 檔，則應將所有選定檔案的資訊整合成「單一標準 Markdown 檔案」輸出。
   - **檔名命名規則**：`[YYYY-MM-DD]_[簡短主題描述].md`（如已是整合檔案，可命名為 `[YYYY-MM-DD]_[專案名稱]整合規劃.md`）。
   - **檔案內容結構**：必須符合下方模板，完成資訊的結構化與初步切片：

```markdown
---
source01: "[第一個原始檔名或網址]"
source02: "[第二個原始檔名或網址，若有複數檔案依此類推：source03、source04...]"
import_date: [YYYY-MM-DD]
category: "Brief / Background"
tags: [[標籤1], [標籤2]]
---

# [專案名稱/主題]

## 1. 原始內容文字大綱 (Raw Content Outline)
[若有多個來源，請以 ### 部分一：[來源A名稱]、### 部分二：[來源B名稱] 分段紀錄各檔案之核心文字大綱，剔除贅詞與格式代碼。若為單一來源則直接撰寫大綱。]

## 2. 核心變數備忘錄 (Core Variable Notes)
[將所有選定來源的關鍵變數合併收攏至此處統一撰寫：]
- 品牌與品類背景：
- 目標受眾與痛點初步描述：
- 預算與管道限制：
- 主要競爭對手：
```

4. **建檔確認回報與自動推進**：
   - 建檔完成後，小D 必須向使用者回報建檔路徑，確認已完成資料切片，然後**自動、無縫地進入 Phase 1**，不可在此處中斷對話。
